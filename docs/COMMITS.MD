# Commit Convention

## Language

**All commit messages must be written in English.**

Using English ensures consistency across the codebase, makes the git history readable for any contributor, and produces clean, professional release notes automatically.

---

## Format

```text
<emoji> <type>: <short description>
```

---

## Commit types

| Type     | Description                        |
| -------- | ---------------------------------- |
| feat     | New feature                        |
| fix      | Bug fix                            |
| docs     | Documentation changes              |
| style    | Formatting, no logic change        |
| refactor | Code refactor                      |
| perf     | Performance improvements           |
| test     | Add or fix tests                   |
| chore    | Maintenance tasks                  |
| build    | Build system changes               |
| security | Security improvements              |
| revert   | Revert changes                     |
| config   | Configuration changes              |
| remove   | Remove files or code               |
| add      | Add files or code                  |
| merge    | Branch merge                       |
| change   | General change (not a new feature) |

---

## Emojis

| Emoji | Type     | Emoji | Type     |
| ----- | -------- | ----- | -------- |
| ✨    | feat     | 🚀    | perf     |
| 🐛    | fix      | ✅    | test     |
| 📝    | docs     | 📦    | chore    |
| 🎨    | style    | 📤    | build    |
| 🔨    | refactor | 🔒    | security |
| ⏪    | revert   | 🔧    | config   |
| 🗑️    | remove   | ➕    | add      |
| 🔀    | merge    | ⚡    | change   |

---

## Examples

```text
✨ feat: add BaseButton component with disabled and loading props
🐛 fix: correct icon alignment in MenuCard when no label is provided
📝 docs: update README with installation steps
🔨 refactor: simplify useSearch hook logic
🗑️ remove: delete obsolete helper file
```

---

## Notes

- Only `✨ feat:` and `🐛 fix:` commits are included in the release notes generated by `release.sh`.
- Keep the description short and specific — mention the component, prop, or behavior affected.
- Use the **imperative mood**: *add*, *fix*, *update*, *remove* — not *added*, *fixes*, *updating*.
